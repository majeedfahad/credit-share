# Pay.dt.sa — Personal Payment Tracker

## Overview
نظام شخصي لتتبع المصروفات مبني بـ Laravel. يستقبل رسائل SMS من بنك الراجحي تلقائياً عبر iPhone Shortcuts → API، يسجل كل عملية دفع، يصنفها تلقائياً حسب اسم التاجر، ويعرض dashboard لمتابعة المصاريف.

## Owner
عبدالمجيد الطويل (مجيد) — مشروع شخصي بالكامل.

## Architecture
- **Backend:** Laravel (PHP) + MySQL
- **Frontend:** Blade templates
- **Hosting:** DigitalOcean (Forge setup)
- **Data Source:** iPhone Shortcuts يقرأ SMS الراجحي → يرسل POST لـ `/api/payments/incoming`
- **Auth:** API key per device (Bearer token) + web login for dashboard

### ⚠️ مشكلة الرصيد في البطاقة الفرعية
عند الدفع بالبطاقة الفرعية (2568)، رسالة البنك تعرض رصيد الفرعية فقط، لكن المبلغ ينخصم من الأساسية والفرعية معاً. يحتاج حل لمزامنة الرصيد الحقيقي للبطاقة الأساسية.

## SMS Format (Al Rajhi)
الرسائل تجي من بنك الراجحي بصيغة نصية عربية، يتم parse-ها في `IncomingPaymentController::parseMessage()`. الحقول المتوقعة:
- `بطاقة:` — رقم البطاقة + نوعها
- `مبلغ:` — المبلغ بالريال
- `لدى:` — اسم التاجر
- `رصيد:` — الرصيد المتبقي
- `في:` — التاريخ والوقت

## Salary Cycle
- الراتب ينزل يوم **25** من كل شهر
- الدورة: من 25 الشهر الحالي إلى 24 الشهر التالي
- يتم إنشاء الدورة تلقائياً عند أول عملية دفع في الفترة

## Auto-Classification
- جدول `merchant_categories` يربط أنماط أسماء التجار بالتصنيفات
- عند تصنيف عملية يدوياً، النظام يحفظ النمط للتصنيف التلقائي مستقبلاً
- التصنيفات: طعام، بقالة، وقود، تسوق، منزل، صحة، مواصلات، ترفيه، تقنية، اشتراكات، تحويل، أخرى

## Family Access
- ViewToken يسمح لأفراد العائلة بمشاهدة مصروفات البطاقة الفرعية (2568) فقط
- مربوط بـ fingerprint للجهاز عند أول استخدام
- لا يوجد وصول عائلي للبطاقة الأساسية (3091)

## Known Issues & TODOs

### 🔴 عاجل: فقدان عمليات عند انقطاع النت
iPhone Shortcuts يفشل في إرسال البيانات عند انقطاع الإنترنت والعملية تضيع.
**المطلوب:** آلية تخزين محلي (offline queue) في الـ Shortcut تحفظ الرسائل الفاشلة وتزامنها تلقائياً عند رجوع الاتصال.

### 🟡 رصيد البطاقة الأساسية عند الدفع بالفرعية
عند الدفع بـ 2568، الرصيد المعروض في الـ SMS هو رصيد الفرعية فقط. البطاقة الأساسية (3091) ما يتحدث رصيدها تلقائياً.
**المطلوب:** عند تسجيل دفعة من الفرعية، خصم المبلغ من رصيد الأساسية أيضاً.

### 🟢 ميزات مخطط لها
- **تقارير أسبوعية عبر تيليقرام:** ملخص المصروفات بالتصنيف + مقارنة بالأسبوع السابق
- **تنبيه حد الميزانية:** إشعار تيليقرام عند الاقتراب من حد معين (مثلاً 80% من الميزانية)
- **تصنيف تفاعلي عبر تيليقرام:** عند عملية غير مصنفة، يرسل سؤال لمجيد يختار التصنيف
- **تقرير شهري:** ملخص نهاية الدورة مع مقارنة بالشهور السابقة
- **تنبيه العمليات الكبيرة:** إشعار فوري لأي عملية تتجاوز مبلغ معين

## Development Notes
- الـ parser يدعم صيغة SMS الراجحي فقط حالياً
- `FailedPayment` model موجود لتسجيل العمليات الفاشلة + retry
- الـ API endpoint `/api/payments/retry` يعيد محاولة العمليات الفاشلة (max 5 retries)
- Forge deployment على DigitalOcean، الـ Raspberry Pi عنده SSH access للسيرفر
